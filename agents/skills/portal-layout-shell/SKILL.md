# Portal Layout Shell

Use this skill when editing layout, navigation, dashboard composition, responsive structure, or overall portal consistency.

## Objective

Ensure the portal feels like one professionally designed product with a stable shell, strong navigation, and a consistent page structure.

## Core layout rules

The portal should use:
- a consistent left sidebar
- a consistent main content region
- a clear page header structure
- consistent section composition
- responsive behavior that is intentionally designed

Avoid:
- random top navigation inside the portal shell
- disconnected page layouts
- each page looking like a different app
- cramped mobile conversions of desktop layouts
- floating sections with no structure

## Sidebar rules

The left sidebar is a core product element.

Requirements:
- maintain a polished sidebar
- keep navigation visually consistent across pages
- make active state obvious
- keep spacing between nav items deliberate
- keep icons or markers aligned
- ensure the sidebar looks premium, not generic admin template

Do not:
- add random duplicate navigation
- create multiple competing nav systems
- place important portal navigation only in page-local controls

## Page shell rules

Each page should follow a shared structure:
1. page intro / heading area
2. summary or key actions
3. main content sections
4. secondary detail sections where appropriate

This structure should remain visually consistent across the portal.

## Dashboard composition

Dashboards should:
- present hierarchy clearly
- group related information together
- use featured cards intentionally
- avoid clutter
- avoid cards of random heights and widths without purpose

Preferred composition:
- welcome/header section
- top summary metrics or highlights
- primary actionable content
- secondary informational content
- recent activity, status, or timeline sections where relevant

## Responsive behavior

Responsiveness must be designed, not accidental.

Requirements:
- desktop should feel spacious and premium
- tablet should preserve hierarchy
- mobile should stack gracefully
- tables should remain usable
- forms should remain readable
- sidebar behavior should be intentional on smaller screens

## Visual cohesion rules

Layout changes must support cohesion:
- consistent maximum widths
- consistent gutters
- consistent section spacing
- repeated patterns should reuse the same structural components
- cards and sections should align cleanly

## Navigation labels

Navigation labels should:
- be clear
- be concise
- be consistent in style and casing
- reflect real destinations

## Shell quality check

Before finalizing:
- verify every portal page looks related
- verify sidebar consistency
- verify heading placement consistency
- verify card spacing consistency
- verify responsive stacking behavior
- verify there is no leftover template clutter

## Output rules

When this skill is used:
- prioritize structural clarity
- improve product feel
- preserve working navigation
- do not create fake destinations
- return complete, usable layout code
