# Portal Finish Pass

Use this skill after implementing portal changes, or whenever the user asks for the portal to feel more finished, professional, polished, premium, or production-ready.

## Objective

Do the final pass that makes the difference between "working" and "well built."

## Final pass checklist

Review and improve:

### Visual consistency
- heading scale
- body text consistency
- card radius consistency
- shadow consistency
- border consistency
- badge consistency
- button sizing consistency
- icon alignment consistency

### Spacing
- page padding
- card padding
- grid gaps
- form spacing
- table cell spacing
- section separation
- sidebar spacing

### States
- loading states
- empty states
- error states
- disabled states
- hover states
- focus states

### Product feel
- remove awkward wording
- remove placeholder text
- remove generic template artifacts
- improve visual hierarchy
- make key actions obvious
- ensure the portal feels intentionally designed

### Responsive quality
- inspect desktop layout
- inspect tablet layout
- inspect mobile stacking
- ensure controls remain usable on smaller screens
- ensure headers and cards do not collapse awkwardly

## Finish standard

The final result should feel:
- calm
- premium
- trustworthy
- cohesive
- production-ready

It should not feel:
- rushed
- template-generated
- inconsistent
- cluttered
- unfinished

## Output rules

When this skill is used:
- perform a true finish pass, not superficial renaming
- improve both aesthetics and usability
- preserve actual working functionality
- return clean final code
